Publishing Documents with Pillar 7.0

Template Management

In the new version of Pillar, we have a new way of managing templates files. Normally, by default, we should specify templates files location in pillar.conf.

Main features:

  • By default, we don't need to specify template files
  • Specify template files per pillar files

Default templates values

Previously for archetypes, all template files were located in templates directory under _support of the project directory. We were then obliged to specify all the path to the mustche files in pillar.conf. Now we refactor archetypes in order to have templates files in related writer directories under templates directory. For example, in book archetype, we have epub, latex, html directories under templates directory. Then now all templates files in those files are named main.mustache instead of different and long names. We just only need to specify the template name main.mustchae in general case and acording to the writer, the file will be resolved from the good directory. We then set that as default values, so we don't specify mustache files in pillar.conf. Now in our pillar.conf we don't have information about template files, only about writers.

Mustache template per pillar file

With these new features, we can have a website or book with pages generated from different mustache templates files. All we have to do is to specify using template property the name of the template files. We put our mustache template in the corresponding directory. Then we just specify the basename of the template file. Error cases

For default templates files, when they are not present, we have a FileNotFound Exception stopping the generation process.
